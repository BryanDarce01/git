# `.gitignore`
Es un archivo de configuración de Git que le dice a Git qué archivos o carpetas **NO** debe rastrear ni incluir en el repositorio.

## Para qué sirve

Se usa principalmente para ignorar archivos que no deberían estar versionados, por ejemplo:

- rchivos temporales

- Archivos generados automáticamente

- Configuraciones locales

- Dependencias

- Archivos sensibles

## Ejemplos típicos:

- node_modules/

- env

- archivos .log

- carpetas build/ o dist/
- archivos del sistema como .DS_Store

## Ejemplo:

``` md

# ignorar archivos de logs
*.log

# ignorar variables de entorno
.env

# ignorar dependencias
node_modules/

# ignorar archivos de compilación
dist/
build/

# ignorar archivos del sistema
.DS_Store

```

### IMPORTANTE

`.gitignore` solo funciona con archivos que Git todavía **NO** está rastreando.

## Dónde se coloca

El archivo debe estar en la raíz del repositorio:

```

proyecto/
│
├─ .git/
├─ .gitignore
├─ src/
├─ README.md
```

Pero también se puede tener `.gitignore` en subcarpetas con reglas específicas.
