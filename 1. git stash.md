#  `git stash`

Los comandos `git stash`, `git stash pop` y `git stash apply` se usan para guardar temporalmente cambios no confirmados (sin hacer commit) en el área de trabajo. 

Son muy útiles cuando necesitas cambiar de rama o actualizar el repositorio sin perder el trabajo actual.

## 1. `git stash`

**Guarda temporalmente los cambios no confirmados** (tanto en el staging area como en el working directory) y **deja el repositorio** limpio (como si se hubiera hecho checkout a HEAD).

### Git mostrará algo como:

``` pgsql
Saved working directory and index state WIP on main: 3f2a8c2 Update README
```

#### Se recomienda ponerle un nombre a cada stash:

``` bash
git stash save "mensaje descriptivo"
```

## 2. `git stash pop`

Recupera el último stash guardado y lo elimina de la lista de stashes.

## 3. `git stash apply`

Aplica los cambios de un stash sin eliminarlo de la lista.

- Esto aplica el stash más reciente, pero lo mantiene guardado (por si se necesita aplicar otra vez o en otra rama).

- También se puede aplicar un stash específico:

``` bash
git stash apply stash@{2}
```

## 4. Ver y gestionar stashes
``` bash
git stash list
```
Ver y gestionar stashes

## 5. Eliminar uno específico

``` bash
git stash drop stash@{0}
```

## 6. Eliminar todos

``` bash
git stash clear
```
