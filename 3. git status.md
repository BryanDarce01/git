# `git status`

Sirve para ver el **estado actual de los archivos** dentro del repositorio **Git**.

## En otras palabras, muestra:

-   Qué archivos han cambiado.

-   Cuáles están listos para el commit (**en staging**).

-   Cuáles no están siendo rastreados por **Git**.

-   En qué rama (**branch**) se está trabajando.

# Resumen de las posibles salidas:

| Estado                               | Significado                                                        |
| ------------------------------------ | ------------------------------------------------------------------ |
| **Untracked**                        | Git no está rastreando ese archivo aún (`git add` para incluirlo). |
| **Modified**                         | Archivo ya rastreado, pero cambió desde el último commit.          |
| **Staged (Changes to be committed)** | Listo para guardarse en el próximo commit.                         |
| **Unmodified**                       | Archivo sin cambios desde el último commit.                        |
